# Architecture

## Layering

```text
UI -> Application -> Domain
                  -> Security
                  -> Infrastructure
```

The application layer coordinates clipboard reads, domain transformations, session state, and model requests. All model-facing content must pass through the security layer before reaching infrastructure backends.

## Prompt Security Rule

The model client must never receive raw user-controlled text directly.

All clipboard-derived or freeform user text is treated as untrusted and must pass through:

1. `InputSanitizer.sanitize`
2. `TrustWrapper.wrap`
3. `PromptBuilder.build`

Trusted prompt sections are limited to:

- static system prompts
- application-generated instructions
- operation metadata generated by code
- cryptographic nonce and time markers

Untrusted sections include:

- clipboard contents
- pasted logs
- URLs
- decoded payloads
- extracted text from artifacts

## Request Flow

```text
raw input
  -> sanitize unicode
  -> wrap as <untrusted_input>
  -> generate nonce/time markers
  -> build trusted prompt policy
  -> format for backend transport
  -> send to hosted API or CLI backend
```

## Security Note

All model-facing prompts must be assembled through a dedicated `PromptBuilder`. User-controlled or clipboard-derived content is treated as untrusted, sanitized to remove Unicode tag blocks and orphaned surrogates, then wrapped in explicit untrusted-input tags before interpolation. Trusted prompt sections include application-generated embedded tokens and cryptographically secure nonce/time markers in line with the SKB-183-style separation pattern described in the design.

## Backends

The application speaks to a `ChatBackend` abstraction rather than a specific provider. Current scaffolded backend families are:

- Bedrock backend for AWS-hosted models
- OpenAI-compatible backend for DeepSeek and Kimi
- CLI agent backends for Claude Code and Codex

Each backend converts a shared `PreparedPrompt` into its transport-specific request shape while preserving the same trust boundaries.
